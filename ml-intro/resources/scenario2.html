Multicolor Classification Scenario

Overview

This scenario focuses on classifying inputs based on their RGB color values into one of multiple categories, such as red, magenta, yellow, white, blue, cyan, and green. Users can adjust the neural network's hyperparameters to improve accuracy.

Interface

  • Input: Users can select an RGB color using the color picker or adjust the R, G, B values manually.
  • Output: The available classes are represented by colors: red, magenta, yellow, white, blue, cyan, and green.
  • Bias: Users can adjust the bias to control the network's constant term.
  • Learning Rate: Controls how much the weights change during training.
  • Activation Functions: Users can select activation functions for both hidden and output layers. Available options include ReLU, Sigmoid, Tanh, and SoftMax.
  • Epochs: Number of iterations for training the network.
  • Run Button: Starts the training process based on the selected parameters.
  • Save/Load: Save the current network state or load a previously saved state.

How the Neural Network Works

The neural network processes the input (R, G, B values) through one hidden layer. The activations are calculated using the selected activation functions. The network learns to associate input colors with their corresponding categories by adjusting weights and biases during training.

Recommendations for Training

  • Learning Rate: Start with a small learning rate (e.g., 0.01) to ensure stable training.
  • Bias: Experiment with different bias values to observe their impact on the network's performance.
  • Epochs: Begin with a low number of epochs (e.g., 10) and gradually increase as needed.
  • Activation Functions: SoftMax is recommended (meaning it will not work properly with any other activation function) for the output layer in classification problems. For hidden layers, try ReLU for faster convergence.
  • Visualization: Observe changes in the network diagram and predictions after training to analyze performance.
